Common Questions

Frequently Asked Questions

How does the Random Forest classifier calculate predictions?
Instead of passing raw pixel values through heavy convolution pipelines, we extract 21 hand joints coordinate vectors (x, y, z parameters) via MediaPipe. We then perform wrist-relative coordinate translations and divide vectors by their L2 norm. This creates a scale-invariant, aspect-ratio-invariant feature dataset which our Random Forest model processes in under 15ms.
Is the model trained to support both left and right hand orientations?
Yes. During dataset preparation, we implemented Horizontal Mirroring data augmentation, reflecting the x-axis values of detected landmarks horizontally relative to the frame width. This trains the ensemble trees to recognize poses identically regardless of hand orientation.
How does the Live Transcription workspace define a completed letter?
To prevent flicker and erroneous classification triggers, the system runs a stable holds prediction queue. A character must be classified continuously with >70% confidence for 4 consecutive frames (~1.2 seconds) before it is committed to the text transcription buffer.
